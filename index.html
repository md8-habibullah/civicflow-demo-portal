Product UI & Feature Proof

Real app screens and what they prove

These screens show how the mobile app and backend dashboard work together: citizens speak, the AI understands the issue, reports can be submitted with GPS, and reviewers can monitor cases from the dashboard.

Protected Backend Dashboard

Admin command center for report review, GPS evidence, and live response monitoring.

The dashboard is the authority-side workspace of CivicFlow AI. When a citizen submits a civic problem or emergency report, the backend stores the report and shows it here for review. It displays the report type, emergency status, source, area, transcript, AI summary, recommended action, and GPS location.

This helps reviewers understand what happened, where it happened, and what action is recommended. The dashboard also separates submitted reports and live tracking so emergency cases can be monitored faster. This proves the app has a working backend review flow, not only a front-end demo.

Submitted Reports Citizen voice reports appear for review.
GPS Evidence Latitude, longitude, accuracy, and Google Maps link.
AI Summary Short explanation of what the citizen reported.
Status Control Reviewers can manage report progress.
Citizen Home UI

Simple voice-first interface for stressful civic and emergency situations.

The home screen is designed around one main action: tap the microphone and speak naturally. Users do not need to fill a long form or search through complicated menus.

The Help button gives quick access to the helpline directory, while Profile, History, and Safety are placed around the screen without distracting from the main voice report flow.

This design is especially useful for elderly users, rural users, or people under pressure because it reduces typing, searching, and manual report steps.

Helpline & Area Recommendation

Category-based help for gas, electricity, water, ambulance, blood bank, and cyber/fraud support.

This screen lets users find national and regional emergency contacts without scrolling through a long mixed list. Users can choose a problem type and use GPS to find the most relevant local or national helpline.

The same logic supports the voice assistant. Direct contact requests return only the matched number, while real incident reports can use location when needed.

  • Category-based help selection
  • GPS-supported area recommendation
  • Direct contact lookup without unnecessary GPS report submission
Risk-Zone Warning

Bangla safety warning when the user enters a known risky area.

The risk-zone warning screen explains the risky area, district, possible safety concerns, and recommended precautions in Bangla. This feature is important because civic safety is not only about reporting after something happens; it can also warn users before they stay too long in a risky place.

The warning is based on structured hotspot data and the user’s GPS location. If the location matches a saved risk zone, the app can show this warning and guide the user to stay alert.

Safety Monitor

Optional GPS-based safety check for nearby risk zones.

The Safety Monitor lets users turn on risk-zone checking while the app is running. Users can also tap “Check Now” to immediately check whether their current location matches a known hotspot.

This screen clearly explains how the feature works: CivicFlow checks GPS location against the saved Bangladesh risk-zone dataset, then sends a warning and safety guidance if a match is found.

  • User-controlled safety monitoring
  • GPS-based risk-zone check
  • Notification and warning screen support
App Identity

Civic safety, voice assistance, and location awareness in one visual identity.

The CivicFlow AI icon combines a shield, microphone, and location marker. The shield represents public safety, the microphone represents voice-first AI interaction, and the location marker represents GPS-based reporting and risk-zone awareness.

This icon can be used across the Android app, Vercel demo portal, video thumbnail, and presentation slides so the project looks consistent and professional.

AI & Architecture

How CivicFlow AI thinks

The current version combines Gemini voice understanding with structured civic retrieval and deterministic safety rules. Emergency actions are protected by rules, not blindly generated by AI.

🎙️ Voice Input Bangla, English, or Banglish spoken report
🧠 Gemini AI Intent and issue understanding
📚 Civic Directory Helpline, contact, blood bank, and risk-zone data
🛡️ Safety Rules Decides when GPS/reporting is allowed
📊 Dashboard Report review, status, and live tracking
